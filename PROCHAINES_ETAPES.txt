📋 LISTE DES ACTIONS
════════════════════════════════════════════════════════════════════════════════

ACTION 1: PRÉPAREZ-VOUS
─────────────────────────────────────────────────────────────────────────
□ Fermez l'application Java (complètement)
□ Assurez-vous que MySQL est en cours d'exécution
□ Ouvrez phpMyAdmin (http://localhost/phpmyadmin)

ACTION 2: EXÉCUTEZ LA MIGRATION
─────────────────────────────────────────────────────────────────────────
□ Sélectionnez votre base de données (pidev ou autre)
□ Cliquez sur l'onglet "SQL"
□ Collez la commande SQL suivante:

ALTER TABLE recolte
ADD COLUMN type_culture VARCHAR(100) NOT NULL DEFAULT 'Non spécifié',
ADD COLUMN localisation VARCHAR(100) NOT NULL DEFAULT 'Non spécifiée';

□ Cliquez sur "Exécuter"
□ Attendez le message de succès

ACTION 3: VÉRIFIEZ LA MODIFICATION
─────────────────────────────────────────────────────────────────────────
□ Toujours dans phpMyAdmin
□ Cliquez sur la table "recolte"
□ Cliquez sur l'onglet "Structure"
□ Vérifiez que vous voyez 6 colonnes:
   □ id_recolte
   □ quantite
   □ date_recolte
   □ qualite
   □ type_culture (NOUVEAU)
   □ localisation (NOUVEAU)

ACTION 4: REDÉMARREZ L'APPLICATION
─────────────────────────────────────────────────────────────────────────
□ Fermez l'IDE (IntelliJ, Eclipse, VS Code, etc.)
□ Attendez 5 secondes
□ Rouvrez l'IDE
□ Redémarrez votre application Java
□ Attendez la connexion: "✅ Connexion à la base de données réussie !"

ACTION 5: TESTEZ
─────────────────────────────────────────────────────────────────────────
□ Menu Principal: Sélectionnez option 3 (Gestion des Récoltes)
□ Sous-menu: Sélectionnez option 1 (Ajouter une récolte)
□ Remplissez les données:
   □ ID culture: 1
   □ Quantité: 6000
   □ Date récolte: 12/02/2026
   □ Qualité: Premium
   □ Type culture: Blé
   □ Localisation: Seliana
□ Cliquez Ajouter
□ Vérifiez le message: "✅ Récolte ajoutée avec succès ! ID: X"
□ SUCCÈS! ✅

❓ QUESTIONS POSSIBLES
════════════════════════════════════════════════════════════════════════════════

Q: Vais-je perdre mes données?
R: Non. Cette migration est non-destructive. Toutes les données existantes
   sont conservées. Les nouvelles colonnes auront une valeur par défaut.

Q: Combien de temps ça prendra?
R: Moins de 1 seconde pour l'exécution SQL. Environ 20 minutes au total
   (incluant redémarrage et test).

Q: Et si j'ai une erreur "Access Denied"?
R: Assurez-vous de vous connecter avec un utilisateur ayant les droits
   ALTER TABLE. Généralement, utilisez "root".

Q: Et si rien ne change?
R: Vérifiez que:
   - Vous avez exécuté la bonne requête
   - Vous avez cliqué sur "Exécuter"
   - Vous vous trouvez dans la bonne base de données

Q: Puis-je annuler la migration?
R: Oui, avec: ALTER TABLE recolte DROP COLUMN type_culture, DROP COLUMN localisation;
   Mais ne faites cela que si vraiment nécessaire.
