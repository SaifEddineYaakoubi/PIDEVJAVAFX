🔍 ANALYSE DE LA CAUSE
════════════════════════════════════════════════════════════════════════════════

DÉSYNCHRONISATION:
┌──────────────────────────────────────────────────────────────────────────┐
│ CODE JAVA (À jour)                │ BASE DE DONNÉES (Obsolète)          │
├──────────────────────────────────┼─────────────────────────────────────┤
│ Recolte.java:                    │ Table recolte:                      │
│ ✅ idRecolte (INT)               │ ✅ id_recolte (INT)                 │
│ ✅ quantite (DOUBLE)             │ ✅ quantite (DOUBLE)                │
│ ✅ dateRecolte (LocalDate)       │ ✅ date_recolte (DATE)              │
│ ✅ qualite (STRING)              │ ✅ qualite (VARCHAR)                │
│ ✅ typeCulture (STRING)          │ ❌ type_culture (MANQUANTE)         │
│ ✅ localisation (STRING)         │ ❌ localisation (MANQUANTE)         │
│                                  │                                     │
│ → 6 attributs                    │ → 4 colonnes seulement              │
└──────────────────────────────────┴─────────────────────────────────────┘

ROOT CAUSE:
   1. Le modèle Recolte.java a été mis à jour (suppression de idCulture)
   2. Les colonnes type_culture et localisation ont été ajoutées au code
   3. La base de données n'a pas été migrée en conséquence
   4. Quand RecolteService.add() essaie d'insérer, il utilise les
      colonnes qui n'existent pas → Erreur MySQL
