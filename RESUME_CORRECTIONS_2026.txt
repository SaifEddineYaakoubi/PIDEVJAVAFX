🎯 LE PROBLÈME (Avant les corrections)
═══════════════════════════════════════════════════════════════════════════════

DESCRIPTION:
  "L'ajout de récolte à partir de l'interface est sauvegardé dans la base
   de données MAIS la liste de récolte ne s'affiche pas dans l'interface"

SYMPTÔMES OBSERVÉS:
  ❌ Application démarre OK
  ❌ Dashboard s'affiche OK
  ❌ Clic sur "🌾 Récoltes" → TableView vide
  ❌ Ajout d'une récolte → Données sauvegardées en base (vérifiées)
  ❌ MAIS: Aucune récolte visible dans le TableView
  ❌ Difficult to debug sans messages

CAUSE IDENTIFIÉE:
  La colonne "Actions" du TableView était déclarée avec un type incorrect:
  - Type déclaré: TableColumn<Recolte, String>
  - Type requis: TableColumn<Recolte, Void>

  Cela a causé une incohérence dans la liaison de données qui a empêché
  le rendu correct de la table entière.

═══════════════════════════════════════════════════════════════════════════════
✅ LES SOLUTIONS (Ce qui a été corrigé)
═══════════════════════════════════════════════════════════════════════════════

CORRECTION 1: Type de la colonne d'action
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Fichier: src/main/java/org/example/pidev/controllers/RecolteController.java
Ligne: 36

AVANT:
  @FXML
  private TableColumn<Recolte, String> colAction;

APRÈS:
  @FXML
  private TableColumn<Recolte, Void> colAction;

RAISON:
  La colonne d'actions ne lie aucune propriété du modèle (elle génère
  seulement des boutons). Le type Void est correct car aucune donnée
  de propriété n'est liée.


CORRECTION 2: Type de TableCell pour la colonne d'action
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Fichier: src/main/java/org/example/pidev/controllers/RecolteController.java
Méthode: setupTableColumns()

AVANT:
  colAction.setCellFactory(col -> new TableCell<Recolte, String>() {
      @Override
      protected void updateItem(String item, boolean empty) {
          // ...
      }
  });

APRÈS:
  colAction.setCellFactory(col -> new TableCell<Recolte, Void>() {
      @Override
      protected void updateItem(Void item, boolean empty) {
          // ...
      }
  });

RAISON:
  Alignement du type avec la déclaration de la colonne.
