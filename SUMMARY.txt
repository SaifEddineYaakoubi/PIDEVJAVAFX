📋 PROBLÈME ORIGINAL
═══════════════════════════════════════════════════════════════════════════════

❌ "l'ajout de recolte a partir de l'interface est sauvgargé dans la base
   de donné mais la liste de recolte ne s'affiche pas dans l'interface"

═══════════════════════════════════════════════════════════════════════════════
✅ SOLUTION APPLIQUÉE
═══════════════════════════════════════════════════════════════════════════════

CAUSE RACINE IDENTIFIÉE:
  La colonne "Actions" du TableView était déclarée avec le type incorrect.

  Code problématique:
    @FXML private TableColumn<Recolte, String> colAction;  // ❌ Incorrect
    colAction.setCellFactory(col -> new TableCell<Recolte, String>() { ... });

  Code corrigé:
    @FXML private TableColumn<Recolte, Void> colAction;    // ✅ Correct
    colAction.setCellFactory(col -> new TableCell<Recolte, Void>() { ... });

EXPLICATIONS:
  • La colonne d'actions ne lie aucune propriété du modèle Recolte
  • Elle utilise seulement setCellFactory() pour générer des boutons
  • Le type Void est correct car aucune donnée n'est liée
  • Avec le type String, JavaFX attendait une propriété à lier
  • Cette incohérence empêchait le rendu correct de la table
