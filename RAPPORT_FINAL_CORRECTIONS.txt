🎯 PROBLÈME IDENTIFIÉ
═══════════════════════════════════════════════════════════════════════════════

TITRE: "La liste de récolte ne s'affiche pas dans l'interface"

DESCRIPTION:
  Après l'ajout d'une récolte via le formulaire, les données sont bien
  sauvegardées en base de données (MySQL), mais la liste n'est pas
  affichée dans le TableView de l'interface JavaFX.

SYMPTÔMES:
  • Le bouton "➕ Ajouter Récolte" fonctionne
  • Le formulaire de saisie s'affiche correctement
  • Les données sont sauvegardées en base (vérifiées via MySQL)
  • MAIS: Le TableView reste vide, aucune récolte ne s'affiche

═══════════════════════════════════════════════════════════════════════════════
🔍 ANALYSE DE LA CAUSE
═══════════════════════════════════════════════════════════════════════════════

PROBLÈME RACINE IDENTIFIÉ:
  La colonne "Actions" du TableView était déclarée avec un type incorrect.

DÉTAILS TECHNIQUES:

  ❌ CODE AVANT (INCORRECT):
     @FXML
     private TableColumn<Recolte, String> colAction;

     colAction.setCellFactory(col -> new TableCell<Recolte, String>() {
         @Override
         protected void updateItem(String item, boolean empty) {
             // Génération de boutons...
         }
     });

  ✅ CODE APRÈS (CORRECT):
     @FXML
     private TableColumn<Recolte, Void> colAction;

     colAction.setCellFactory(col -> new TableCell<Recolte, Void>() {
         @Override
         protected void updateItem(Void item, boolean empty) {
             // Génération de boutons...
         }
     });

EXPLICATION:
  • La colonne d'actions NE lie aucune propriété du modèle Recolte
  • Elle utilise UNIQUEMENT setCellFactory() pour générer des boutons
  • Avec un type <String>, JavaFX attend une propriété String à lier
  • Cela crée une incohérence qui peut empêcher le rendu correct
  • Le type correct est <Void> car aucune propriété n'est liée
