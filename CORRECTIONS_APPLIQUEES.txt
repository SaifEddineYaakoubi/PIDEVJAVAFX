✅ CORRECTIONS EFFECTUÉES
═══════════════════════════════════════════════════════════════

1️⃣  CORRECTION DU TYPE DE COLONNE D'ACTION
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Fichier: RecolteController.java

   ❌ AVANT:
      @FXML
      private TableColumn<Recolte, String> colAction;

      // Dans setupTableColumns():
      colAction.setCellFactory(col -> new TableCell<Recolte, String>() { ... });

   ✅ APRÈS:
      @FXML
      private TableColumn<Recolte, Void> colAction;

      // Dans setupTableColumns():
      colAction.setCellFactory(col -> new TableCell<Recolte, Void>() { ... });

   🔍 RAISON:
      La colonne d'action génère des boutons via setCellFactory()
      et ne lie aucune propriété du modèle. Le type doit être
      TableColumn<Recolte, Void> et non String.


2️⃣  AMÉLIORATION DU DEBUGGING DANS loadRecoltes()

   Fichier: RecolteController.java

   ✅ MODIFICATIONS:
      - Ajout de messages println() détaillés
      - Affichage du nombre de récoltes chargées
      - Affichage de chaque récolte pour débogage
      - Vérification que tableRecoltes n'est pas null
      - Stack trace en cas d'erreur

   🔍 UTILITÉ:
      Permet d'identifier rapidement si:
      - Les données sont en base de données
      - Les données sont chargées en mémoire
      - La table est correctement initialisée
